Auftrag 1.2: Grundlagen Cloud-Computing

  • Wie ist der Begriff Cloud entstanden? Wieso heisst es Cloud?

    • Der Ursprung kommt hier aus der Symbolisierung von netzwerken in der das Internet zumeist als Wolke dargestellt wurde.

  • Wie wird der Begriff Cloud definiert, z.B. gemäss NIST?

    • Es werden fünf wesentliche merkmale vom NIST beachtet um die Cloud zu definieren, nähmlich: On-Demand Self-Service (Bedarfsgerechter Selbstbedienungszugriff): Nutzer können eigenständig und bedarfsgerecht auf Computing-Ressourcen zugreifen, wie zum Beispiel Serverleistung, Speicherplatz oder Anwendungen, ohne dabei auf Anbieter angewiesen zu sein. Broad Network Access (Breiter Netzwerkzugriff): Die Ressourcen sind über das Netzwerk verfügbar und können von unterschiedlichen Geräten (z. B. PCs, Laptops, Smartphones) und Betriebssystemen aus erreicht werden. Resource Pooling (Ressourcenpool): Die Ressourcen des Cloud-Anbieters werden in einem Pool zusammengefasst und dynamisch an verschiedene Kunden zugewiesen. Die genaue physische Lage der Ressourcen ist für die Nutzer transparent. Rapid Elasticity (Schnelle Skalierbarkeit): Die Ressourcen können schnell und flexibel entsprechend dem aktuellen Bedarf hoch- oder herunterskaliert werden. Dies ermöglicht eine effiziente Nutzung und Anpassung an sich ändernde Anforderungen. Measured Service (Messbare Dienstleistung): Die Nutzung der Ressourcen wird überwacht, kontrolliert und abgerechnet. Die Abrechnung kann beispielsweise auf der Basis der genutzten Rechenleistung, Speicherkapazität oder Nutzungszeit erfolgen.

  • Welches sind die 5 Merkmale einer Cloud?

    • Die 5 merkmale der Cloud lauten wie oben beschrieben: Breiter Netzwerkzugriff, Ressourcenpool, Schnelle Skalierbarkeit, Messbare Dienstleistung, Bedarfsgerechter Selbstbedienungszugriff

  • Welche Cloud Dienstleistungen kennen Sie?

    • Die bekanntesten Cloud Dienstleidungen sind das Betreiben von Servern z.b. als Webserver oder im algemeinen Virtuelle Maschnienen, dazu kommen Datenbanken und Cloud Storage lösungen bei AWS heisst dies S3.

  • Welche Cloud Anbieter kennen Sie?

    • Schweizer Cloud Anbieter sind infomaniak, green.ch, und Cyberlink um einige zu nennen. International währen dies unternemen wie Microsoft und Amazon die Azure und AWS betreiben.

  • Was sind Cloud Service Modelle?

    • Unter Cloud Service Modellen versteht mann, begriffe wie IaaS, Paas oder Saas diese beschreiben im fall IaaS, Infrastructure as a service hier wird eine gesammte infrastuktur als Dienstleistung angeboten. So kann betriebssystem und Kernel vom Konsumenten selbständig ausgewählt werden. IaaS wird genutzt wenn Virtuelle Maschnienen von einem Cloud Anbieter eingesetzt werden.

  • Weshalb soll ich Dienste aus der Cloud beziehen? Was sind die Vorteile?

    • Cloud Dienste können bezogen werden wen keine eingene Hardware angeschafft werden möchte, sowohl schnelle skalierbarkeit wichtig ist für das Unternehmen.

  • Wie werden technologische Beiträge in der Cloud geteilt bzw. zur Verfügung gestellt?

    • Es existieren sogenannte Cloud Marktplätze die von Anbietern wie Azure und AWS zur verfügung gestelt werden, dort können vorgefertigte Cloud Lösungen wie Container oder Virtuelle Maschienen abgerufen und deployed werden.

Auftrag 2.1: Dienstleistungen von Cloud Anbietern vergleichen

  • Microsoft (Azure)

    • Angaben zum Provider

    Gründungsjahr: Microsoft Azure wurde im Jahr 2010 eingeführt, obwohl Microsoft bereits vorher Cloud-basierte Dienste angeboten hat. Firmensitz: Microsoft Corporation hat ihren Hauptsitz in Redmond, Washington, USA. Umsatz: Im Geschäftsjahr 2021 erzielte Microsoft jedoch einen Umsatz von 168,1 Milliarden US-Dollar.

    Angaben zur Cloud und den Diensten:

    Mehr als 200 Cloud-Dienste und Lösungen. Diese umfassen Computing, Speicherung, Datenbanken, Netzwerke, AI, Analyse, IoT und security.

    Einfache virtuelle Maschine (IaaS): In Azure wird dieser Dienst als "Azure Virtual Machines" bezeichnet. Er ermöglicht die Bereitstellung und Verwaltung von virtuellen Maschinen in der Cloud, auf denen Benutzer ihre eigenen Betriebssysteme und Anwendungen ausführen können. Plattform für Applikationen (PaaS): Hier bietet Azure den Dienst "Azure App Service". Es handelt sich um eine vollständig verwaltete Plattform, auf der Entwickler Web- und Mobilanwendungen erstellen, bereitstellen und skalieren können, ohne sich um die zugrunde liegende Infrastruktur kümmern zu müssen. Datenbankdienste: Azure bietet verschiedene Datenbankdienste, darunter "Azure SQL Database" (relationale Datenbank), "Azure Cosmos DB" (NoSQL-Datenbank), "Azure Database for MySQL" (MySQL-Datenbank) und "Azure Database for PostgreSQL" (PostgreSQL-Datenbank). Funktion als Dienst (FaaS): Azure Functions ermöglicht die Entwicklung serverloser Anwendungen, bei denen der Fokus auf der Ausführung von einzelnen Funktionen basierend auf Ereignissen liegt, ohne dass eine gesamte Infrastruktur bereitgestellt oder verwaltet werden muss. Speicherplatzdienst: Der Speicherplatzdienst von Azure heißt "Azure Blob Storage". Er bietet skalierbaren und kostengünstigen Speicherplatz für unstrukturierte Daten wie Texte, Bilder, Videos und mehr.

    Angaben zu den Kosten:

    ca. 30 US-Dollar und 60 US-Dollar liegen, je nach Region und Nutzungsdauer.
